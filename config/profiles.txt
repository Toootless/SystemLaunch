# Webpage Launcher Profile Configuration
# 
# SYSTEM MONITORS & LAYOUTS:
# Monitor 1 → DISPLAY6 - 3840x1620 [Split Left-Right]: 2 sections for unlimited tabs each
# Monitor 2 → DISPLAY7 - 3840x2160 [Grid 2x3]: 6 sections for unlimited tabs each
# Monitor 3 → DISPLAY1 - 5760x3240 [Full Screen]: Unlimited chrome tabs
# Monitor 4 → DISPLAY2 - 5760x3240 [Full Screen]: Unlimited chrome tabs
# Monitor 5 → DISPLAY3 - 7680x2160 [Chrome Groups Left-Right]: 2 sections for chrome groups
#
# Physical Layout:
#     Monitor1       Monitor3
#  Monitor5  Monitor2  Monitor4
#
# FORMAT: monitor,position,location_id,app_type,target[,skip_positioning]
# 
# monitor: 1-5 (monitor number, see mapping above)
# position: Entry sequence number (1, 2, 3, ...). Unlimited entries per monitor.
# location_id: Visual location - determines where to position the window:
#   Monitor 1 (left-right): left, right
#   Monitor 2 (2x3 grid): top-left, top-center, top-right, bottom-left, bottom-center, bottom-right
#   Monitor 3 (full): center
#   Monitor 4 (full): center
#   Monitor 5 (groups): left, right
# app_type: 'chrome' for browser tabs or 'program' for executables
# target: URL for chrome, or executable path for programs
# skip_positioning: (Optional, 6th field) Set to 'true', '1', 'yes', or 'skip' to disable window positioning for this app.
#                   Useful for 3D graphics apps (like Bambu Studio) that interfere with window enumeration.
# NOTE: For full-screen monitors (3, 4), all tabs will open at full resolution. 
#       For split/grid monitors, location_id determines which section the window opens in.
#
# Example entries (unlimited per monitor):

# 1,2,right,chrome,https://www.github.com
# 1,3,left,chrome,https://www.example.com  # Another tab in left section
#

#
